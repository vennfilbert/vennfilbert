Personal Portfolio Website - Project Plan

This is the master checklist for building and launching the personal portfolio website.

Phase 1: Blueprint & Foundation

  • Finalize the website blueprint: all sections and content decided.
  • Write the initial text for the "About Me" and "Projects" sections.
  • Create the `vennfilbert.github.io` public repository on GitHub.
  • Find and download a suitable HTML/CSS starter template.
  • Set up the project locally in VSCode and initialize Git.
  • Customize the basic elements of the template (colors, fonts, name, headline).

Phase 2: Core Build

  • Build the HTML structure and CSS layout for the "Projects" section.
  • Populate the project cards with details from your resume.
  • Build the "Contact" section.
  • Build the "Skills" section.
  • Thoroughly test the website for responsiveness on a mobile device.
  • Proofread all text content for typos and grammatical errors.

Phase 3: Refine & Deploy

  • Write the final, professional `README.md` file for this repository.
  • Take a high-quality screenshot of the finished website to include in the README.
  • Push the final, completed code to the main branch of the GitHub repository.
  • Verify that the website is live and working correctly at `https://vennfilbert.github.io`.

Phase 4: Post-Launch

  • Add the website URL to your resume's contact information.
  • Add the website URL to your LinkedIn profile (contact info and "Featured" section).
  • Add the website URL to your GitHub profile bio and README.
  • Share a post on LinkedIn announcing your new portfolio website.
